# DataTranslation

## 概要

DataTranslationは、卒業研究で制作した「リアルワールドメタバースにおけるデータ授受システム」のUnityフロントエンド実装です。

本システムでは、VR空間に参加するリモートユーザと、MRデバイスを装着して現実空間に参加するユーザが、同一の共有空間内で画像データをやり取りできることを目的としています。

特に、チャットツールや外部のファイル共有サービスに切り替えるのではなく、VR/MR空間内のインタラクションによって、特定の相手へ自然にデータを渡すことを目指しました。

## 背景

リアルワールドメタバースでは、現実空間と仮想空間を接続し、現地ユーザと遠隔ユーザが同じ場にいるように協働することが想定されます。
このような環境では、会話や共同作業の途中で、画像や資料などのデータを特定の相手に共有したい場面が発生します。

しかし、既存のファイル共有方法では、外部アプリケーションやGUI操作に意識が移り、会話や作業の流れを妨げる可能性があります。
そこで本研究では、VR/MR空間内の身体動作や空間的な指定を用いたデータ授受手法を実装し、ユーザ間での自然なデータ共有を検討しました。

## このリポジトリについて

本リポジトリは、卒業研究で制作したシステムのうち、Unityで実装したフロントエンド部分を含んでいます。

主に以下の処理を担当しています。

* VR/MRモードの切り替え
* Photonを用いた複数ユーザの同期
* ログイン処理
* APIサーバからの画像一覧取得
* S3上の画像データの取得・表示
* 共有する画像の選択
* 共有相手の選択
* Hover方式、Ray方式、GUI方式によるデータ授受
* ハンドジェスチャやコントローラ入力による操作

バックエンド側では、FastAPI、PostgreSQL、Docker、AWS S3を用いて、ユーザ情報、ファイル情報、共有状態などを管理しています。
本リポジトリではUnityフロントエンド部分を公開しています。バックエンドはFastAPI、PostgreSQL、Docker、AWS S3を用いて構成しており、認証情報や環境依存情報を含むため、本リポジトリには含めていません。

## 主な機能

### 1. ユーザログイン

ユーザ名とパスワードを用いてAPIサーバへログインします。
ログイン後、ユーザ名をUnity側で保持し、ファイル一覧取得やデータ共有処理に利用します。

### 2. VR/MRモード選択

ユーザはVRモードまたはMRモードを選択して参加できます。
VRユーザは仮想空間から参加し、MRユーザは現実空間を基準としたパススルー環境で参加します。

### 3. 画像一覧の取得・表示

ログイン中のユーザが閲覧可能な画像データをAPIサーバから取得し、Unity上のCanvasに一覧表示します。
画像データはAPI経由で取得したURLを用いてダウンロードし、空間内UIに表示します。

### 4. データ授受手法

本システムでは、以下の3種類のデータ授受手法を実装しました。

#### Hover方式

共有したい画像をカードとして生成し、相手の受取カードに近づけることでデータ授受を行う方式です。
「手渡し」に近い操作感を目指し、相手に渡したことを空間的に確認しやすい点を重視しました。

#### Ray方式

レイを用いて共有相手を指定し、選択した相手へデータを送信する方式です。
相手に近づかずに共有できるため、距離がある場面でも素早く指定できることを目的としています。

#### GUI方式

共有相手を一覧UIから選択し、ボタン操作によってデータを共有する方式です。
従来のGUIに近い操作であり、相手を明示的に選択できる確実性を重視しました。

## バックエンド構成について

本リポジトリは、卒業研究で制作したシステムのうち、Unityフロントエンド部分を公開したものです。

### バックエンドの役割

バックエンドでは、主に以下の処理を担当しています。

* ユーザログイン

* ユーザごとの閲覧可能ファイルの管理
* ユーザ間の共有状態の登録
* Unityフロントエンドへの画像URL・ファイル情報の提供

Unity側では、APIを通じてバックエンドから画像一覧や共有状態を取得し、VR/MR空間内のUIやインタラクションに反映しています。

### Unity側から利用する主なAPI

Unityフロントエンドでは、以下のようなAPIを利用しています。

```text
POST /login
ユーザ名・パスワードを送信し、ログイン処理を行う。

GET /files
ログイン中のユーザが閲覧可能な画像ファイル一覧を取得する。

GET /files/{id}
指定した画像ファイルの情報を取得する。

POST /share
選択した画像ファイルを、指定したユーザへ共有する。
```

これらのAPIを通じて、Unity側では画像一覧の表示、共有する画像の選択、共有相手へのデータ授受処理を行っています。

### データベース設計

バックエンドでは、主に以下の情報をデータベースで管理しています。

* Users：ユーザ情報
* Files：画像ファイル情報
* Shareable：ユーザが共有可能なファイル情報
* Readable：ユーザが閲覧可能なファイル情報

これにより、各ユーザがどのファイルを閲覧できるか、どのファイルを他ユーザへ共有できるかを管理しています。

### セキュリティ上の理由による公開範囲

本システムのバックエンドには、データベース接続情報、S3関連の設定、API接続先など、環境依存の情報が含まれます。
そのため、本公開リポジトリではUnityフロントエンド部分を中心に掲載し、認証情報や環境変数、実験用データなどは含めていません。

## 主要スクリプト

### `GameManager.cs`

Photonへの接続、ルーム参加、VR/MRプレイヤーの生成、参加ユーザリストの管理を行います。
VRモードとMRモードに応じて、使用するカメラやプレイヤーオブジェクトを切り替えています。

### `ApiClient.cs`

FastAPIサーバと通信するための共通クライアントです。
GET/POSTリクエストを行い、ログイン、ファイル取得、共有処理などで利用します。

### `LoginManager.cs`

ログイン処理を担当します。
入力されたユーザ情報をAPIサーバへ送信し、ログイン成功時にプレイシーンへ遷移します。

### `ImageLoader.cs`

APIサーバから画像一覧を取得し、取得した画像をUnity上のCanvasに表示します。
ファイルIDやファイル名をUI要素に保持し、共有対象の選択に利用します。

### `DataSharing.cs`

データ共有処理の中心となるスクリプトです。
選択されたファイルIDと共有相手のユーザIDを管理し、選択中の共有方式に応じてHover方式、Ray方式、GUI方式の処理を切り替えます。

### `HoverMethod.cs`

Hover方式の処理を担当します。
共有カードをPhoton経由で生成し、右手の位置に追従させることで、手渡しに近いデータ授受を実現します。

### `ControllerRaySelect.cs`

Ray方式の処理を担当します。
右手またはコントローラの方向へRayを飛ばし、命中した相手のユーザIDを取得して共有処理につなげます。

### `GUIMethod.cs`

GUI方式の処理を担当します。
参加中のユーザ一覧を取得し、自分以外のユーザを共有相手候補としてUI上に表示します。

### `HandGestureDetector.cs`

XR Handsを用いて、手のトラッキング情報を取得するスクリプトです。
MR環境におけるハンドジェスチャ操作の基礎部分として利用しています。

### `ReceiveID.cs`

受取側カードにユーザIDを保持し、共有カードとの接触時に共有相手を判定します。
Hover方式における「誰に渡したか」を判定するために利用します。

## 操作方法

### VRユーザ

* ファイル一覧表示：左コントローラのXボタン
* ファイル選択：右コントローラのトリガー
* 共有決定：Aボタンまたはトリガー
* Ray方式での相手指定：右コントローラのRay
* Hover方式での共有カード操作：右手に追従するカードを相手の受取カードへ近づける

### MRユーザ

* ファイル一覧表示：左手のグー操作
* ファイル選択：Poke操作
* 共有決定：右手のグー操作
* キャンセル：チョキ操作
* Hover方式での共有：手の動きに追従するカードを相手の受取カードへ近づける

## 実装上の工夫

### VR/MRで異なる入力方法への対応

VRユーザはコントローラ、MRユーザはハンドトラッキングを用いるため、同じ機能でも適した入力方法が異なります。
本システムでは、VRとMRで操作デバイスが異なっても、同じ目的を達成できるように入力設計を行いました。

### 3種類の共有手法を切り替えられる設計

Hover方式、Ray方式、GUI方式を同一システム上で切り替えられるように実装しました。
これにより、同じタスクに対して複数のインタラクション手法を比較できる構成にしています。

### Photonを用いた空間内オブジェクト同期

共有カードやユーザアバタなど、複数ユーザ間で共有する必要があるオブジェクトはPhotonを用いて生成・同期しています。
これにより、VRユーザとMRユーザが同じ空間内でデータ授受を行えるようにしました。

### API・DB連携を前提とした共有処理

画像データや共有状態をUnity内だけで完結させず、APIサーバ・データベースと連携して管理する構成にしました。
これにより、ユーザごとの閲覧権限や共有履歴を扱えるようにしています。
